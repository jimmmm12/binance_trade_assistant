## 当前自动化说明

- 软件启动后不会自动开始 24 小时交易。
- 用户点击自动交易页的“启动自动”后，系统才会按当前设置的间隔循环扫描。
- 当前默认自动扫描间隔为 5 分钟。
- 自动交易默认优先模拟和风控校验；真仓下单仍需要 API、环境开关、确认文本和风险检查全部通过。

## 下一阶段计划

### 1. 计划质量评分系统升级（提前优先做）

目标：让每个交易计划不仅有方向，还能解释“为什么值得做”和“为什么不该做”。

计划改进：

- 拆分评分维度：趋势结构、成交量、ATR 波动风险、RSI 拥挤度、资金费率、盘口/流动性、当前仓位冲突、强平安全垫。
- 输出计划等级：可真仓、只模拟、只观察、禁止下单。
- 扫描结果新增三列：推荐理由、主要风险点、建议操作。
- 低评分计划自动锁定真仓按钮，只允许模拟或观察。
- 把评分结果写入自动交易日志，便于后续复盘和参数自适应。

### 2. 自动交易状态机

目标：把自动化从“定时执行按钮逻辑”升级为清晰的交易生命周期。

状态设计：

- 空仓观察
- 发现机会
- 生成计划
- 等待确认 / 自动模拟
- 已开仓
- 持仓管理
- 减仓 / 移动止损
- 平仓复盘

重点解决：

- 防止重复下单。
- 防止卖出数量超过模拟仓或真实仓。
- 防止计划和右侧快速下单栏不同步。
- 网络错误后能恢复到正确状态。

### 3. WebSocket 实时行情

目标：价格、K 线、成交量从 REST 轮询升级为 WebSocket 推送。

计划改进：

- 实时订阅持仓币种和候选币种价格。
- 断线自动重连。
- 行情超过指定秒数未更新时禁止真仓下单。
- REST API 只保留账户、持仓、下单和补数据用途。

### 4. 持仓自动管理

目标：开仓后自动执行更接近真实交易系统的保护逻辑。

计划改进：

- 到 1R 后止损移动到成本价附近。
- 到 1.5R 后减仓 20%-30%。
- 到 2R 后继续移动止损并保留尾仓。
- 反向破结构时提前减仓。
- 强平安全垫不足时强提醒或强制锁定加仓。

### 5. 自动化日志和复盘页

目标：每一次自动扫描、拒绝、下单、减仓和平仓都有记录。

计划改进：

- 记录当时的价格、ATR、RSI、成交量、资金费率、评分、仓位和风控结果。
- 增加复盘页：按币种、策略模式、时间、盈亏过滤。
- 用复盘数据反向优化参数自适应引擎。
