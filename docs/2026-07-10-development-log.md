## 当前自动化说明

- 软件启动后不会自动开始 24 小时交易。
- 用户点击自动交易页的“启动自动”后，系统才会按当前设置的间隔循环扫描。
- 当前默认自动扫描间隔为 5 分钟。
- 自动交易默认优先模拟和风控校验；真仓下单仍需要 API、环境开关、确认文本和风险检查全部通过。

## 下一阶段计划

### 1. 计划质量评分系统继续升级

目标：把当前规则评分继续升级成可回测、可复盘、可调权重的评分引擎。

计划改进：

- 加入盘口深度和滑点估算。
- 加入大盘状态过滤，避免逆 BTC/ETH 主趋势硬做。
- 把仓位占用、同方向风险暴露、相关币种风险纳入评分。
- 用复盘结果动态调整评分权重。

### 2. 自动交易状态机增强

目标：把自动化从“定时执行按钮逻辑”升级为清晰的交易生命周期。

重点解决：

- 把状态持久化到数据库，软件重启后恢复自动化阶段。
- 区分模拟仓状态机和真实仓状态机。
- 加入状态超时处理，例如计划过期、行情断流、仓位读取失败。

### 3. WebSocket 实时行情

目标：价格、K 线、成交量从 REST 轮询升级为 WebSocket 推送。

计划改进：

- 实时订阅持仓币种和候选币种价格。
- 断线自动重连。
- 行情超过指定秒数未更新时禁止真仓下单。
- REST API 只保留账户、持仓、下单和补数据用途。

### 4. 持仓自动管理

目标：开仓后自动执行更接近真实交易系统的保护逻辑。

计划改进：

- 到 1R 后止损移动到成本价附近。
- 到 1.5R 后减仓 20%-30%。
- 到 2R 后继续移动止损并保留尾仓。
- 反向破结构时提前减仓。
- 强平安全垫不足时强提醒或强制锁定加仓。

### 5. 自动化日志和复盘页

目标：每一次自动扫描、拒绝、下单、减仓和平仓都有记录。

计划改进：

- 当前已记录价格、ATR、RSI、成交量、资金费率、评分、动作和风控结果。
- 增加复盘页：按币种、策略模式、时间、盈亏过滤。
- 用复盘数据反向优化参数自适应引擎。
