# 3. 每个启动文件是干什么的

## 3.1 run_scan_all.bat

用途：

```text
扫描现货 + U本位合约
```

适合：

```text
你想同时看现货和合约机会
```

结果：

```text
reports\latest.md
reports\latest.csv
```

## 3.2 run_scan_futures.bat

用途：

```text
只扫描U本位合约
```

适合：

```text
你想看做多/做空候选
```

这是你现在最常用的文件。

## 3.3 run_scan_spot.bat

用途：

```text
只扫描现货
```

适合：

```text
你只想买现货，不想碰合约
```

用途：


```text
UNIUSDT
合约做多
入场 3.25
止损 3.12
目标 3.38
本金 1000U
单笔风险 1%
杠杆 2倍
```

输出位置：

```text
reports\plan_UNIUSDT_long.md
```

## 3.5 run_order_dry_run_example.bat

用途：

```text
模拟下单
```

特点：

```text
不会真实买入
不会真实卖出
只会显示 dry_run: true
```

适合：

```text
测试命令格式是否正确
```

## 3.6 run_order_live_TEMPLATE.bat

用途：

```text
真下单模板
```

注意：

```text
这个文件可能真实下单。
不要直接乱点。
使用前必须先右键编辑。
```

# 4. 怎么看 latest.md 报告

打开：

```text
reports\latest.md
```

你会看到类似字段：

```text
市场
交易对
方向
分数
最新价
24h涨跌
成交额(百万)
RSI 1h
RSI 4h
成交量倍数
24h动量
3日动量
资金费率
备注
```

## 4.1 市场

```text
合约：U本位合约
现货：币安现货
```

## 4.2 方向

```text
做多：观察上涨机会
做空：观察下跌机会
```

注意：  
看到“做多”不等于马上买，看到“做空”不等于马上空。

## 4.3 分数

```text
分数越高，越值得观察
```

但分数不是下单指令。

分数高只能说明：

```text
趋势、量能、动量、资金费率等条件比较匹配
```

还必须自己等入场条件。

## 4.4 RSI

```text
RSI偏高：可能过热
RSI偏低：可能超跌
```

简单理解：

```text
RSI > 75：追多要小心
RSI < 25：追空要小心
```

## 4.5 成交量倍数

```text
成交量倍数越高，说明最近成交越活跃
```

例如：

```text
1.00：最近成交量和之前差不多
2.00：最近成交量约为之前2倍
5.00：明显放量
```

## 4.6 资金费率

只对合约有意义。

简单理解：

```text
资金费率偏高：多头可能拥挤
资金费率偏低或负数很大：空头可能拥挤
```

## 4.7 备注

常见备注：

```text
偏多观察：等回踩支撑不破再考虑，不要直接追高
偏空观察：等反弹到压力位失败再考虑，不要暴跌后追空
```

这是最重要的一列。

# 5. 正确使用流程

每天使用顺序：

```text
1. 双击 run_scan_futures.bat
2. 打开 reports\latest.md
3. 看偏多/偏空名单
4. 选1-3个标的，不要看太多
5. 自己画支撑压力
6. 用 run_plan_example.bat 的格式计算仓位

```

新手推荐：

```text

但只做模拟
不要高杠杆
不要追涨杀跌
```

# 6. 怎么计算交易方案

打开：

```text
run_plan_example.bat
```

右键选择：

```text
编辑
```

你会看到类似：

```bat
python -m trade_assistant.main plan --symbol UNIUSDT --market futures --side long --entry 3.25 --stop 3.12 --target 3.38 --equity 1000 --risk-pct 1 --leverage 2
```

参数解释：

```text
--symbol UNIUSDT       交易对
--market futures       合约
--market spot          现货
--side long            做多
--side short           做空
--entry 3.25           入场价
--stop 3.12            止损价
--target 3.38          目标价
--equity 1000          本金
--risk-pct 1           单笔最多亏本金的1%
--leverage 2           2倍杠杆
```

## 6.1 做多示例

```bat
python -m trade_assistant.main plan --symbol UNIUSDT --market futures --side long --entry 3.25 --stop 3.12 --target 3.38 --equity 1000 --risk-pct 1 --leverage 2
```

要求：

```text
做多必须是：止损价 < 入场价 < 目标价
```

## 6.2 做空示例

```bat
python -m trade_assistant.main plan --symbol ENAUSDT --market futures --side short --entry 0.076 --stop 0.080 --target 0.070 --equity 1000 --risk-pct 1 --leverage 2
```

要求：

```text
做空必须是：目标价 < 入场价 < 止损价
```

## 6.3 输出怎么看

它会输出：

```text
最多亏损金额
建议数量
名义仓位
需要保证金
到止损亏损百分比
到目标盈利百分比
加杠杆后到止损约亏多少
加杠杆后到目标约赚多少
```

# 8. 真下单怎么用

真下单文件：

```text
run_order_live_TEMPLATE.bat
```

## 8.1 第一步：右键编辑

右键：

```text
run_order_live_TEMPLATE.bat
```

选择：

```text
编辑
```

## 8.2 第二步：填写 API

找到：

```bat
set BINANCE_API_KEY=PUT_YOUR_API_KEY_HERE
set BINANCE_API_SECRET=PUT_YOUR_API_SECRET_HERE
```

改成你的 API：

```bat
set BINANCE_API_KEY=你的API_KEY
set BINANCE_API_SECRET=你的API_SECRET
```

注意：

```text
API Key 不要开提现权限。
只开交易权限。
```

## 8.3 第三步：打开第二重开关

找到：

```bat
set BINANCE_ENABLE_LIVE_TRADING=true
```

这一行就是第二重开关。

如果它是：

```bat
set BINANCE_ENABLE_LIVE_TRADING=true
```

说明允许真下单。

如果你想关闭真下单，改成：

```bat
set BINANCE_ENABLE_LIVE_TRADING=false
```

## 8.4 第四步：确认第三重开关

命令最后必须有：

```bat
--allow-live --confirm LIVE_TRADING_CONFIRMED
```

完整例子：

```bat
python -m trade_assistant.main order --market spot --symbol UNIUSDT --side BUY --quantity 1 --type MARKET --allow-live --confirm LIVE_TRADING_CONFIRMED
```

如果没有：

```bat
--allow-live --confirm LIVE_TRADING_CONFIRMED
```

程序不会真下单。

## 8.5 三重开关总结

真下单必须同时满足：

```text
1. API Key 和 Secret 填了
2. set BINANCE_ENABLE_LIVE_TRADING=true
3. 命令里有 --allow-live --confirm LIVE_TRADING_CONFIRMED
```

少一个条件，就不会真下单。

# 10. 安全关停方法

如果你不想让它真下单，有三种方法。

## 方法1：关闭环境变量

把：

```bat
set BINANCE_ENABLE_LIVE_TRADING=true
```

改成：

```bat
set BINANCE_ENABLE_LIVE_TRADING=false
```

## 方法2：删除确认参数

删除命令最后的：

```bat
--allow-live --confirm LIVE_TRADING_CONFIRMED
```

## 方法3：删除 API

把：

```bat
set BINANCE_API_KEY=你的API_KEY
set BINANCE_API_SECRET=你的API_SECRET
```

改回：

```bat
set BINANCE_API_KEY=PUT_YOUR_API_KEY_HERE
set BINANCE_API_SECRET=PUT_YOUR_API_SECRET_HERE
```

最稳的方法：

```text
三种都做。
```

# 12. 每次交易前检查

下单前必须回答：

```text
我为什么买/卖？
入场价是多少？
止损价是多少？
目标价是多少？
这笔最多亏多少钱？
如果连续亏3笔，我会不会停手？
这个币我真的懂吗？
我是在按计划交易，还是因为害怕错过？
```

答不出来，就不要下单。

# 13. 常见问题

## 13.1 为什么 bat 文件是英文？

因为 Windows 批处理文件遇到中文容易乱码，甚至命令无法运行。

所以：

```text
bat 文件名和窗口提示保留英文
报告和说明文件使用中文
```

## 13.2 报告里的分数是不是越高越可以买？

不是。

分数高只是说明值得观察。

还要看：

```text
有没有到支撑/压力位置
RSI是不是过热
有没有放量
BTC大环境好不好
止损能不能接受
```

## 13.3 为什么不直接自动全仓买？

因为那不是交易助手，是赌博按钮。

这个工具的目标是：

```text
帮你筛选
帮你计算
帮你控制风险
帮你记录
```

不是替你乱冲。

## 13.4 可以以后做更好用的界面吗？
